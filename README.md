

**基于 MuJoCo MPC 的汽车仪表盘可视化系统**

---

## 一、实验环境与配置

### 1. 硬件与系统环境

* 操作系统：Ubuntu 22.04 LTS
* CPU：x86_64 架构
* 内存：≥ 8GB
* 显卡：支持 OpenGL 3.3

### 2. 软件环境

* 编译器：g++
* 构建工具：CMake
* 版本控制：Git
* 图形库：OpenGL
* 物理引擎：MuJoCo

---

## 二、系统整体设计

### 1. 系统组成

本系统由以下三部分组成：

1. **物理仿真模块（MuJoCo）**

   * 负责车辆模型运动、速度与位置计算

2. **数据获取模块（C++）**

   * 从 `mjData` 中读取速度、位置等仿真数据
   * 转换为仪表盘所需数据格式

3. **仪表盘渲染模块（OpenGL）**

   * 将仿真数据实时绘制为 2D 汽车仪表盘（HUD）

---

## 三、场景建模与仿真运行

### 1. 场景说明

* 使用 MuJoCo 的 MJCF（XML）格式
* 构建一个简单车辆模型：

  * 车身
  * 四个车轮
  * 自由关节

### 2. 场景运行方式

```bash
./bin/mjpc --task=SimpleCar
```

 场景可正常加载并运行，车辆在仿真环境中运动。

---

## 四、仿真数据获取

### 1. 数据来源

从 MuJoCo 的 `mjData` 结构中读取：

* `data->qpos`：车辆位置
* `data->qvel`：车辆速度
* `data->qacc`：车辆加速度
* `data->ctrl`：控制输入

### 2. 获取内容

| 数据项 | 说明           |
| --- | ------------ |
| 速度  | 由 x、y 方向速度合成 |
| 转速  | 根据车速进行模拟映射   |
| 油量  | 随时间递减        |
| 温度  | 与转速关联变化      |

### 3. 数据提取实现说明

* 在仿真主循环中，每一步调用数据更新函数
* 数据实时传递给仪表盘渲染模块

---

## 五、仪表盘实现

### 1. 实现方式

* 使用 OpenGL **2D 正交投影**
* 作为 HUD 覆盖在 3D 仿真画面之上
* 不影响原有 MuJoCo 渲染流程

### 2. 实时性说明

* 仿真每一步更新数据
* 仪表盘每一帧根据最新数据重绘
* 实现数据与画面同步变化
